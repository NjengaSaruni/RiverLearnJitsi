# RiverLearn Jitsi Meet - Self-Hosted Video Conferencing

A complete self-hosted Jitsi Meet setup using Docker Compose, configured for the `riverlearn.co.ke` domain. This provides a fully functional video conferencing solution with JWT authentication integration.

## 🎯 Features

- ✅ **Complete Jitsi Meet Stack**: Prosody, Jicofo, JVB, and Web interface
- ✅ **JWT Authentication**: Integrates with RiverLearn Course Organizer
- ✅ **SSL/TLS**: Automatic Let's Encrypt certificates
- ✅ **Custom Branding**: No Jitsi watermarks or branding
- ✅ **Recording**: Meeting recording capabilities
- ✅ **Live Streaming**: YouTube/Twitch streaming support
- ✅ **Breakout Rooms**: Virtual breakout room functionality
- ✅ **Mobile Support**: Works on all devices and browsers

## 🏗️ Architecture

The setup includes all essential Jitsi Meet components:

- **Prosody**: XMPP server for signaling
- **Jicofo**: Conference focus component
- **JVB**: Jitsi Videobridge for media routing
- **Web**: Jitsi Meet web interface

## 🚀 Quick Start

### Option 1: Google Cloud Engine (Recommended)

```bash
# 1. Clone the repository
git clone https://github.com/NjengaSaruni/RiverLearnJitsi.git
cd RiverLearnJitsi

# 2. Edit deployment script with your GCP project ID
nano deploy-to-gce.sh
# Change "your-project-id" to your actual GCP project ID

# 3. Run the deployment
./deploy-to-gce.sh
```

### Option 2: Manual Setup

```bash
# 1. Clone the repository
git clone https://github.com/NjengaSaruni/RiverLearnJitsi.git
cd RiverLearnJitsi

# 2. Generate secrets and update configuration
./setup.sh

# 3. Start services
docker-compose up -d
```

### 2. Configure DNS

Add these DNS records pointing to your server IP:

```
jitsi.riverlearn.co.ke                    → YOUR_SERVER_IP
conference.jitsi.riverlearn.co.ke         → YOUR_SERVER_IP
focus.jitsi.riverlearn.co.ke              → YOUR_SERVER_IP
guest.jitsi.riverlearn.co.ke              → YOUR_SERVER_IP
jitsi-videobridge.jitsi.riverlearn.co.ke  → YOUR_SERVER_IP
```

### 3. Start Services

```bash
docker-compose up -d
```

### 4. Check Status

```bash
# View logs
docker-compose logs -f

# Check running containers
docker-compose ps
```

## 🔧 Configuration

### Environment Variables

Key configuration is in `jitsi.env`:

- `XMPP_DOMAIN`: Your main domain (jitsi.riverlearn.co.ke)
- `DOCKER_HOST_ADDRESS`: Your server's public IP
- `LETSENCRYPT_EMAIL`: Email for SSL certificates
- `ENABLE_AUTH`: Enable JWT authentication
- `ENABLE_RECORDING`: Enable meeting recording

## 🐛 Troubleshooting

### Common Issues

1. **DNS not resolving**: Wait 24-48 hours for DNS propagation
2. **SSL certificate issues**: Check Let's Encrypt logs in `web/letsencrypt/`
3. **Port conflicts**: Ensure ports 80, 443, 10000, 5222, 5280, 5347 are available
4. **Firewall issues**: Open required ports on your server

### Logs

```bash
# All services
docker-compose logs

# Specific service
docker-compose logs web

# Follow logs in real-time
docker-compose logs -f web
```

### Reset Everything

```bash
# Stop and remove all containers
docker-compose down

# Remove volumes (WARNING: This deletes all data)
docker-compose down -v

# Start fresh
docker-compose up -d
```
